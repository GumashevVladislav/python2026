# Задание 31.3. Словари и множества. паттерны использования, типичные ошибки
## Блок урока
Множества и проверки уникальности — Учимся отсеивать дубликаты, быстро проверять принадлежность и работать с операциями над наборами.

## Условие задачи
Хакатон проверяет команды, участвующие в офлайн и онлайн форматах. Определи пересечение и команды, которым нужно выдать дополнительные доступы. Организаторы вовремя отправят инструкции.

## Пример входных данных
```python
set_a = {'gamma', 'beta', 'lambda'}
set_b = {'sigma', 'lambda', 'zeta'}
```

## Пример выходных данных
```text
В обоих форматах: {'lambda'}
Нужны доступы: {'sigma', 'zeta'}
```

## Критерии успешного выполнения
- Создаётся два множества без дубликатов.
- Пересечение и разности вычисляются встроенными операторами (&, -, |).
- Выводятся два осмысленных множества согласно задаче.
- Лишние преобразования в списки не используются.

## Подсказки
- Используй операции &, |, - напрямую, так код получается короче.
- После вычислений напечатай подписи, чтобы понять, какое множество что означает.
- Если хочешь сохранить порядок отображения, можешь обернуть множество в sorted для вывода.

## Частые ошибки и ограничения
- Не превращай множества в списки и обратно без необходимости.
- Не дублируй элементы вручную, Python сам сбрасывает повторы.
- Не забывай пояснить, что именно отображаешь в консоли.
