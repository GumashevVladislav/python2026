# Задание 29.3. Коллекции уровня «джун+». списки, срезы, вложенные структуры

## Блок урока
Мутация и методы списков. Учимся добавлять, переставлять и удалять элементы, чтобы гибко обновлять список задач без создания копий.

## Условие задачи
Ты готовишь дорожную карту домашней мастерской по ремонту гитар:
1. Создай список первичных работ.
2. Добавь через `extend` задачи, которые прислал клиент в электронном письме.
3. Вставь на позицию №3 проверку струн, чтобы выполнить её до покраски.
4. Удали по значению работу, которую клиент отменил.
5. Перемести первый элемент в конец с помощью `pop(0)` и `append`.
6. Выведи итоговый список с нумерацией и кратким комментарием к каждому пункту.

## Пример входных данных
```python
jobs = ["шлифовка корпуса", "замена колков", "окраска"]
additional = ["проверка электроники", "полировка грифа"]
```

## Пример выходных данных
```
1. замена колков — подготовка к настройке
2. проверка струн — исключить износ
3. окраска — финишное покрытие
4. проверка электроники — перед сборкой
5. полировка грифа — финальный блеск
6. шлифовка корпуса — перенос в конец
```

## Критерии успешного выполнения
- Использованы минимум три различных метода изменения списка
- Выполнены вставка по индексу, удаление по значению и перемещение элемента
- Итоговый список распечатан после всех операций
- Работа ведётся с исходным списком без создания новых копий

## Подсказки
- После `extend` новые элементы сразу находятся в конце списка
- Индекс 2 соответствует третьей позиции
- Перед удалением убедись, что значение точно присутствует
- Нумерованный вывод удобен через `enumerate(list, start=1)`

## Частые ошибки и ограничения
- Не модифицируй список, пока одновременно его перебираешь
- Следи, чтобы индексы не выходили за пределы длины
- Не применяй конкатенацию списков для перемещения элементов
- Не вызывай `remove`, если не проверил наличие элемента
