# Задание 29.4. Коллекции уровня «джун+». списки, срезы, вложенные структуры

## Блок урока
Вложенные структуры. Тренируем построение и разбор списков словарей, чтобы систематизировать связанные данные и выбирать нужные фрагменты.

## Условие задачи
Ты ведёшь базу команд хакатона:
1. Создай список словарей `teams` с ключами `name`, `track`, `members` (список словарей с `role` и `person`), `score`.
2. Выведи перечень команд в формате `Название — трек, баллы X`, затем перечисли участников с их ролями.
3. Собери множество всех ролей, встречающихся у команд.
4. Построй словарь `track -> средний балл`, усреднив значения `score` для команд одного трека.
5. Отфильтруй команды, где есть роль из переменной `focus_role`, и выведи их названия.

## Пример входных данных
```python
teams = [
    {
        "name": "ByteWave",
        "track": "fintech",
        "members": [
            {"role": "Backend", "person": "Артём"},
            {"role": "Analyst", "person": "Лена"}
        ],
        "score": 87
    },
    {
        "name": "CareAI",
        "track": "health",
        "members": [
            {"role": "ML", "person": "Дина"},
            {"role": "Designer", "person": "Илья"}
        ],
        "score": 92
    }
]
focus_role = "Designer"
```

## Пример выходных данных
```
ByteWave — трек fintech, баллы 87
  - Backend: Артём
  - Analyst: Лена
CareAI — трек health, баллы 92
  - ML: Дина
  - Designer: Илья
Все роли: {'Backend', 'Analyst', 'ML', 'Designer'}
Средние баллы по трекам: {'fintech': 87, 'health': 92}
Команды с ролью Designer: ['CareAI']
```

## Критерии успешного выполнения
- Данные о командах описаны списком словарей с вложенными списками участников
- Вывод формируется через циклы без хардкода индексов
- Уникальные роли и средние баллы считаются программно
- Фильтр по роли не модифицирует исходный список `teams`

## Подсказки
- Для множества ролей используй `roles.add(member["role"])`
- Для среднего по треку удобен словарь `track_stats`, где хранятся сумма и количество
- Проверяй наличие роли через вложенный цикл `for member in team["members"]`

## Частые ошибки и ограничения
- Не редактируй сам список команд при фильтрации
- Следи за правильным обращением к ключам вложенных словарей
- Не копируй код для каждого трека вручную. расчёты делай в цикле
- Не усредняй баллы статически. вычисляй из данных
