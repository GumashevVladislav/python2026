# Практическое задание 35.2

## Задания

### Описание задачи
В предыдущем блоке ты научился создавать функции, передавать в них аргументы и получать результат через `return`. Сейчас мы переходим к важной теме: управление аргументами. Ты будешь работать со значениями по умолчанию для параметров и научишься правильно передавать аргументы — как позиционно, так и по имени. Это поможет избежать частых ошибок, особенно при работе с изменяемыми типами данных, такими как списки.

Твоя задача — написать функцию для управления списком задач (to-do list). Функция должна безопасно добавлять задачи в список, используя значения по умолчанию, и позволять гибко указывать приоритет и категорию задачи с помощью позиционных и именованных аргументов.

### Пример входных и выходных данных

**Пример 1: Безопасное добавление задачи (использование None-паттерна)**
```python
# Вызов функции
list1 = add_task("Купить молоко")
print(list1)  # ['Купить молоко']

list2 = add_task("Прочитать книгу")
print(list2)  # ['Прочитать книгу']

# Обрати внимание: list1 и list2 — разные списки!
```

**Пример 2: Добавление задачи в существующий список**
```python
my_tasks = ["Зарядить телефон"]
updated_list = add_task("Сделать уроки", tasks=my_tasks)
print(updated_list)  # ['Зарядить телефон', 'Сделать уроки']
```

**Пример 3: Использование именованных аргументов для приоритета и категории**
```python
# Порядок аргументов: сначала позиционные, потом именованные
task1 = add_task("Позвонить маме", priority="high")
print(task1)  # ['Позвонить маме | Приоритет: high']

task2 = add_task("Записаться к врачу", category="здоровье", priority="medium")
print(task2)  # ['Записаться к врачу | Категория: здоровье | Приоритет: medium']

# Ошибка! Нельзя передавать именованный аргумент перед позиционным
# add_task(category="работа", "Написать отчет")  # SyntaxError
```

**Пример 4: Комбинированный сценарий**
```python
shared_list = []
add_task("Купить хлеб", tasks=shared_list, category="продукты")
add_task("Сдать проект", tasks=shared_list, priority="high")
print(shared_list)
# ['Купить хлеб | Категория: продукты', 'Сдать проект | Приоритет: high']
```

### Критерии проверки и ограничения
1. **Обязательные требования:**
   - Функция должна называться `add_task`.
   - Первый параметр — `description` (описание задачи, строка).
   - Второй параметр — `tasks` (список задач). По умолчанию должен быть `None`.
   - Третий параметр — `priority` (приоритет, строка). По умолчанию `"normal"`.
   - Четвертый параметр — `category` (категория, строка). По умолчанию `None`.

2. **Логика работы:**
   - Если `tasks` равен `None`, внутри функции создается новый пустой список.
   - Задача добавляется в список в формате строки.
   - Если указан `priority` (кроме значения по умолчанию), к описанию добавляется `" | Приоритет: {priority}"`.
   - Если указана `category` (не `None`), к описанию добавляется `" | Категория: {category}"`.
   - Функция возвращает измененный список `tasks`.

3. **Ограничения:**
   - Не используй `*args`, `**kwargs`, распаковку, кортежи или словари для аргументов.
   - Не используй глобальные переменные внутри функции.
   - Строго соблюдай порядок параметров: `description`, `tasks`, `priority`, `category`.

### Решение задачи (псевдокод)
1. Объяви функцию `add_task` с четырьмя параметрами в строгом порядке:
   - `description` (строка, обязательный)
   - `tasks` (список, по умолчанию `None`)
   - `priority` (строка, по умолчанию `"normal"`)
   - `category` (строка, по умолчанию `None`)

2. Внутри функции:
   - Если `tasks` равен `None`:
     - Создать новый пустой список
   - Иначе:
     - Использовать переданный список `tasks`

3. Сформировать итоговое описание задачи:
   - Начать с `description`
   - Если `priority` не равен `"normal"`:
     - Добавить к описанию `" | Приоритет: "` и значение `priority`
   - Если `category` не равен `None`:
     - Добавить к описанию `" | Категория: "` и значение `category`

4. Добавить сформированную строку в список `tasks`

5. Вернуть список `tasks` как результат функции

### Рекомендации
- Вынести в отдельную функцию: Всю логику по добавлению задачи нужно реализовать в одной функции `add_task`. Дополнительные функции не требуются.
- PEP8:
  - Названия параметров должны быть в `snake_case` (например, `task_description`).
  - После запятой в списке параметров ставь пробел: `def add_task(description, tasks=None):`
