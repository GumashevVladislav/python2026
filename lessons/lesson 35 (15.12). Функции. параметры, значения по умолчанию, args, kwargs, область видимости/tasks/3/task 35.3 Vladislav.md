# Практическое задание 35.3

Это задание относится к третьему блоку урока, посвященному гибким функциям. В предыдущих блоках ты научился создавать функции с позиционными и именованными аргументами, а также использовать значения по умолчанию. Сейчас мы переходим к работе с переменным числом аргументов. Ты будешь использовать `*args` и `**kwargs` для создания универсальных функций, а также освоишь технику распаковки списков и словарей при их вызове. Эти навыки позволят писать код, который легко адаптируется к разному количеству входных данных.

## Задания

### Описание задачи
Ты пишешь программу для обработки данных о студентах. Информация о студенте может включать разное количество характеристик: имя, возраст, оценки по предметам, дополнительные заметки и т.д. Твоя задача — создать функцию `create_student_profile`, которая сможет гибко собирать эту информацию.

Функция должна:
1.  Принимать обязательные аргументы: `name` (имя) и `student_id` (номер студенческого билета).
2.  Принимать любое количество дополнительных **позиционных** аргументов, которые будут интерпретироваться как оценки студента (целые числа).
3.  Принимать любое количество дополнительных **именованных** аргументов (например, `city="Казань"`, `hobby="шахматы"`), которые будут сохранены как дополнительная информация.
4.  Возвращать словарь с ключами: `"name"`, `"id"`, `"grades"` (список оценок) и `"info"` (словарь с дополнительной информацией).

Также тебе нужно создать вторую функцию `print_student_summary`, которая будет красиво выводить основную информацию о студенте. Эту функцию нужно будет вызвать, используя **распаковку словаря**.

### Пример входных и выходных данных

**Пример 1: Базовый вызов**
```python
# Вызов функции
profile = create_student_profile("Анна", 12345, 85, 90, 78, city="Москва", group="A-101")
print(profile)
```
**Ожидаемый вывод:**
```python
{
    'name': 'Анна',
    'id': 12345,
    'grades': [85, 90, 78],
    'info': {'city': 'Москва', 'group': 'A-101'}
}
```

**Пример 2: Вызов с распаковкой для печати**
```python
# Создаем профиль
student_data = create_student_profile("Иван", 67890, 95, 88, specialization="Физика")

# Определяем функцию для печати
def print_student_summary(student_name, student_id, **details):
    print(f"Студент: {student_name}")
    print(f"ID: {student_id}")
    if details:
        print("Дополнительно:", details)

# Вызываем функцию print_student_summary, распаковав словарь student_data
# Нужно передать name, id и все остальное как именованные аргументы
print_student_summary(student_data['name'], student_data['id'], **student_data['info'])
```
**Ожидаемый вывод в консоли:**
```
Студент: Иван
ID: 67890
Дополнительно: {'specialization': 'Физика'}
```

### Критерии проверки и ограничения
1.  Функция `create_student_profile` должна быть объявлена с параметрами `name`, `student_id`, `*args` и `**kwargs`.
2.  Все позиционные аргументы, переданные после `student_id`, должны быть собраны в список `grades`.
3.  Все именованные аргументы, переданные после `*args`, должны быть собраны в словарь `info`.
4.  Функция должна возвращать словарь с описанной структурой.
5.  В коде должен быть пример вызова `print_student_summary` с использованием распаковки (`**`) словаря `info`.
6.  **Запрещено** использовать темы из будущих блоков: `global`, замыкания, обсуждение области видимости (кроме базового понимания, что переменные внутри функции — локальные).

### Решение задачи (псевдокод)
1.  Объяви функцию `create_student_profile` с параметрами: `name`, `student_id`, `*args`, `**kwargs`.
2.  Внутри функции создай пустой словарь `profile`.
3.  Запиши в `profile` по ключу `"name"` значение параметра `name`.
4.  Запиши в `profile` по ключу `"id"` значение параметра `student_id`.
5.  Преобразуй `args` (кортеж) в список и запиши его в `profile` по ключу `"grades"`.
6.  Запиши словарь `kwargs` в `profile` по ключу `"info"`.
7.  Верни словарь `profile`.
8.  Объяви функцию `print_student_summary`, которая принимает `student_name`, `student_id` и `**details`.
9.  Внутри функции напечатай имя и ID студента.
10. Если словарь `details` не пустой, напечатай его содержимое.
11. Создай профиль студента, вызвав `create_student_profile` с тестовыми данными.
12. Для вывода информации вызови `print_student_summary`, передав ей имя и ID из созданного профиля, а также распаковав словарь `info` с помощью `**`.

### Рекомендации
-   Вынести в отдельную функцию: Основную логику по созданию профиля студента уже инкапсулирует `create_student_profile`. Функцию `print_student_summary` также стоит оставить отдельной, так как она решает конкретную задачу форматированного вывода, которую в будущем можно будет легко изменить или повторно использовать.
-   PEP8:
    -   Используй осмысленные имена переменных и параметров (`student_id` вместо `sid`, `details` вместо `d`).
    -   После запятой при перечислении параметров функции или аргументов в вызове ставь пробел: `def func(a, b, *args, **kwargs):`.
