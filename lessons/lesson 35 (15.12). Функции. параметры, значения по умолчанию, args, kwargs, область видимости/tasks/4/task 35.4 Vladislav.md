# Практическое задание 35.4

## Задания

### Описание задачи
В предыдущих блоках ты научился создавать функции, передавать в них аргументы разными способами и возвращать результат. Теперь важно понять, где "живут" переменные, которые ты используешь. В этом блоке мы разберем **область видимости** — правила, которые определяют, где переменная доступна для чтения и изменения.

Твоя задача — написать программу, которая моделирует простой счетчик посещений для разных разделов сайта. Нужно отслеживать общее количество посещений сайта (глобальная переменная) и количество посещений каждого конкретного раздела (локальные переменные внутри функций). Ты должен явно управлять доступом к глобальной переменной и убедиться, что функции работают с правильными данными.

**Контекст:**
У нас есть веб-сайт с двумя разделами: "Новости" и "Блог". Каждый раз, когда пользователь заходит в раздел, нужно:
1. Увеличить общий счетчик посещений сайта.
2. Увеличить счетчик посещений этого конкретного раздела.
3. Вернуть актуальное количество посещений раздела.

**Требования:**
1.  Объяви глобальную переменную `total_visits` для общего счетчика.
2.  Напиши функцию `visit_news()`. Внутри нее:
    *   Объяви локальную переменную `news_visits` (счетчик для раздела "Новости").
    *   Увеличь на 1 глобальную переменную `total_visits`.
    *   Увеличь на 1 локальную переменную `news_visits`.
    *   Верни текущее значение `news_visits`.
3.  Напиши функцию `visit_blog()`. Внутри нее:
    *   Объяви локальную переменную `blog_visits` (счетчик для раздела "Блог").
    *   Увеличь на 1 глобальную переменную `total_visits`.
    *   Увеличь на 1 локальную переменную `blog_visits`.
    *   Верни текущее значение `blog_visits`.
4.  В основной части программы вызови функции несколько раз в разном порядке и выведи результаты.

### Пример входных и выходных данных
**Пример вызова программы:**
```python
# Это пример того, как может выглядеть основной код
print("Первое посещение Новостей:", visit_news())
print("Первое посещение Блога:", visit_blog())
print("Второе посещение Новостей:", visit_news())
print("Общее количество посещений сайта:", total_visits)
```

**Ожидаемый вывод в консоли:**
```
Первое посещение Новостей: 1
Первое посещение Блога: 1
Второе посещение Новостей: 1
Общее количество посещений сайта: 3
```

**Пояснение к примеру:**
*   `visit_news()` вызвана дважды, но каждый раз возвращает `1`. Это потому, что `news_visits` — **локальная переменная**. Она создается заново при каждом вызове функции и "забывается" после ее завершения.
*   `visit_blog()` вызвана один раз и вернула `1` по той же причине.
*   `total_visits` равна `3`, потому что это **глобальная переменная**. Каждый вызов любой из функций увеличивал ее значение, и это изменение сохранялось.

### Критерии проверки и ограничения
*   В решении **обязательно** используется ключевое слово `global` для изменения глобальной переменной внутри функций.
*   Счетчики разделов (`news_visits`, `blog_visits`) должны быть объявлены как локальные переменные внутри соответствующих функций.
*   Нельзя использовать параметры функций для передачи счетчиков разделов или общего счетчика.
*   Программа должна выводить результаты, аналогичные примеру, при последовательных вызовах.

### Решение задачи (псевдокод)
1.  Создать глобальную переменную для общего счетчика посещений, начальное значение 0.
2.  Определить функцию для посещения раздела "Новости":
    *   Объявить, что внутри функции мы будем изменять глобальную переменную общего счетчика.
    *   Создать локальную переменную для счетчика посещений новостей, начальное значение 0.
    *   Увеличить глобальный счетчик на 1.
    *   Увеличить локальный счетчик новостей на 1.
    *   Вернуть значение локального счетчика.
3.  Определить функцию для посещения раздела "Блог" (логика аналогична функции для новостей).
4.  В основной части программы:
    *   Вызвать функцию для новостей и вывести возвращенное значение.
    *   Вызвать функцию для блога и вывести возвращенное значение.
    *   Снова вызвать функцию для новостей и вывести возвращенное значение.
    *   Вывести значение глобального счетчика.

### Рекомендации
*   Вынести в отдельную функцию: В этой задаче выносить логику в дополнительные функции не требуется. Основная цель — наглядно показать разницу между локальной и глобальной областью видимости, используя простые функции.
*   PEP8:
    *   Имена функций должны быть в `snake_case` (например, `visit_news`, `visit_blog`).
    *   После объявления функции с помощью `def` оставляй две пустые строки перед следующим блоком кода (например, перед следующей функцией или основной программой).
