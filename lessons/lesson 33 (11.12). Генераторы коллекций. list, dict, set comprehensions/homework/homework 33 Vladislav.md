## Задача 1. Повторение темы — «monitors»
- **Блок:** list comprehension.
- **Описание:** дан список словарей `monitors = [{"model": "A1", "fps": 75}, ...]`. Собери список строк `"Модель MODEL поддерживает FPS"` только если `fps ≥ 120`.
- **Критерии:** одно выражение list comprehension, исходный список не меняется, формат строки строго `"Модель X поддерживает Y"`.

## Задача 2. Закрепление практики — «checkouts»
- **Блок:** list comprehension с условным форматированием.
- **Описание:** `checkouts = [("Антон", 45000, "Москва"), ...]`. Требуется получить список `"Антон купил на 45000₽"` только для городов, начинающихся на `М`. Для сумм ≥ 60000 выводить `60000K`. Для имён на `а/я` писать «купила».
- **Критерии:** один comprehension, условие по городу и сумме в одном выражении, корректный выбор глагола, исходные данные не трогаем.

## Задача 3. Интеграция — «service stats»
- **Блок:** dict comprehension.
- **Описание:** список `services = [{"name": "API", "requests": 1800, "fail": 3}, ...]`. Сформируй словарь `service_health`, где значение `"ok"` если доля неудачных запросов `< 5%`, иначе `"alert"`. Формула: `fail / requests <= 0.05`. Включай только сервисы с `requests ≥ 1000`.
- **Критерии:** один dict comprehension, фильтр после выражения, тернарный оператор внутри значения.

## Задача 4. Подготовка — «city logs»
- **Блок:** set comprehension.
- **Описание:** список строк `logs = ["Moscow:login", "kazan:update", "perm:logout", " :login"]`. Нужен набор городов в нижнем регистре, где действие `login` или `update`, игнорируя строки с пустым городом или неправильным форматом.
- **Критерии:** одно выражение set comprehension, проверка количества сегментов и непустого города, `strip()` и `lower()` применяются внутри comprehension.
